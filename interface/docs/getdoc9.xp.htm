Alex Tsiberev wrote in message <8e8pbs$30ib$1@adm.sci-nnov.ru>...
>What really the service VPICD_Virtualize_IRQ does? Does it sets your ISR as
>the exclusive interrupt handler (1) or includes it into existing chain of
>ISRs (2)? If (1), is any another service doing (2) ?
  
Treat it as doing #1 for the given IRQ level.
 
If you need to form a chain of Hardware Interrupt Procedures (in VPICD language), then you can do so by hooking the VPICD_Virtualize_IRQ service and playing with fields in the VID structure. It is an old and well established technique, presented in public at least as long ago as 1994, so you should have any trouble finding references.
