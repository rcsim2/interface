For most Win32 functions implemented by KERNEL32, some parameter validation is performed on entry. A search of archives for this newsgroup may reveal a detailed specification of what validation the kernel performs for DeviceIoControl. If the following conflicts with that, then believe the archive - because if I remember posting the message, I probably took more trouble than usual to get it right.
 
If an input buffer is given (meaning non-null address and non-zero size), then the last byte in the buffer and the first byte in the buffer and every byte at multiples of 1000h bytes into the buffer must be readable.
 
If an output buffer is given (meaning non-null address and non-zero size), then the last byte in the buffer and the first byte in the buffer and every byte at multiples of 1000h bytes into the buffer must be writable.
 
If a non-null address is given for a dword that is to receive the number of bytes of output, then the dword at that address must be writable.
 
If a non-null address is given for an OVERLAPPED structure, then the bytes at offsets 00h and 13h at that address (in other words, the first and last bytes of the OVERLAPPED structure) must be readable.
