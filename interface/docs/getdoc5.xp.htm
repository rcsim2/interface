Matt A. wrote in message <7lpdaj$6fi$1@news.harvard.net>...
>Before VXLDR (the system component responsible for managing dynamic VxDs)
>can unload a VxD, it must locate information it created when initially
>loading the VxD.  VXDLDR happens to do a case-sensitive search for this
>information using an ALL CAPS version of the VxD's name.
  
This is not quite right. The VXDLDR requires a case-sensitive match, true. That the VXDLDR uses an ALL CAPS version of the VxD's name is false. The VXDLDR matches the given name - that is, of the VxD you want to unload, with upper and lower case as you supply - against whatever names it knows for the loaded VxDs. These names are known from the VxDs' headers, with upper and lower case as supplied (typically) by the linker. In the matter of unloading VxDs, the VXDLDR just acts on what is given: the VXDLDR is not involved in any conversion of any VxD's name to ALL CAPS.
 
To unload a VxD, it is the responsibility of the programmer to supply the VxD module name, as set in the VxD's header. This name need bear no relation to the VxD device name as set by the programmer in the VxD's DDB. It is true that Microsoft's linker always sets a module name in ALL CAPS.
